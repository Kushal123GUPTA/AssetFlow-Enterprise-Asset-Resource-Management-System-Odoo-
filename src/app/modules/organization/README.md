# organization module

## Module purpose

Organization setup: departments, categories, and employees.

## Owned screens

- OrganizationSetupPage

## Allowed responsibilities

- Department, category, and employee directory UI
- Role assignment forms within organization scope

## Files that belong in the module

- `components/`
- `pages/`
- `hooks/`
- `services/`
- `types/`
- `constants/`
- `index.ts`

## External IDs / data it may consume

- Department IDs
- Category IDs
- Employee/user IDs
- Role constants

## Things it must not implement

- Asset registration
- Allocation or booking logic
- Cross-module feature widgets in shared
- Direct imports from other feature modules
- Feature-specific code placed into `shared/`
